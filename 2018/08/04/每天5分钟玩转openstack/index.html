Openstack 虚拟化简介

虚拟化的类型

类型一：Hypervisor直接安装到物理机上，虚拟机在Hypervisor上运行。

代表：Xen、VMWare

类型二：物理机上先安装操作系统，再将Hypervisor作为操作系统的一个程序模块运行。

代表：KVM、VirtualBox、VMWare Workstation

KVM 简介

KVM全称：Kernel-Based Virtual Machine。

KVM是基于Linux内核进行硬件虚拟化的，kvm.ko用于管理CPU和内存，它只关注虚拟机的调度和内存管理两个方面。IO虚拟化，例如存储和网络虚拟化的任务交给Linux内核和Qemu。

Libvirt 简介

Libvirt是管理Hypervisor的工具，它可以管理KVM、Xen、VirtualBox，openstack底层也是由Libvirt进行管理的。

Libvirt包含：后台daemon程序libvirtd；API库；命令行工具virsh。

virsh list –all

virsh start

CPU 虚拟化简介

使用下面的命令查询cpu是否支持虚拟化，如果支持虚拟化会输出vmx或者svm。

egrep -o ‘(vmx|svm)’ /proc/cpuinfo

内存虚拟化简介

KVM通过内存虚拟化共享物理系统内存，动态分配给虚拟机。

存储虚拟化

KVM的存储虚拟化是通过存储池（storage pool）和卷（volume）来管理。

网络虚拟化 - Linux Bridge

简单理解成二层交换机：用来做TCP/IP二层协议交换的设备。

brctl show 用来查看当前的linux bridge的配置

virbr0 是KVM默认创建的bridge，为连接的虚拟网卡提供NAT访问外网的功能。使用NAT的虚拟机可以访问外网，外网无法访问该虚拟机。

网络虚拟化 - VLAN

LAN：Local Area Network 本地局域网。

两台计算机连接到一个交换机时，就处于同一个LAN。

网络虚拟化 - 虚拟交换机

Linux Bridge + VLAN = 虚拟交换机
