kubernetes入门与实战

发表于 2018-08-04

第一章 准备k8s环境

系统环境：centos7 infor版

kubernetes-1 172.16.1.101
kubernetes-2 172.16.1.102
kubernetes-3 172.16.1.103

sed -i "s/^SELINUX=.*/SELINUX=disabled/" /etc/selinux/config 
hostnamectl set-hostname kubernetes-1 --static
hostnamectl set-hostname kubernetes-1 --transient
hostnamectl set-hostname kubernetes-1 --pretty
reboot

sed -i "s/^SELINUX=.*/SELINUX=disabled/" /etc/selinux/config 
hostnamectl set-hostname kubernetes-2 --static
hostnamectl set-hostname kubernetes-2 --transient
hostnamectl set-hostname kubernetes-2 --pretty
reboot

sed -i "s/^SELINUX=.*/SELINUX=disabled/" /etc/selinux/config 
hostnamectl set-hostname kubernetes-3 --static
hostnamectl set-hostname kubernetes-3 --transient
hostnamectl set-hostname kubernetes-3 --pretty
reboot

安装docker、etcd 、kubernetes

yum -y install docker etcd kubernetes

修改kubenetes配置文件

sed -i "s/--admission-control=NamespaceLifecycle,NamespaceExists,LimitRanger,SecurityContextDeny,ServiceAccount,ResourceQuota/--admission-control=NamespaceLifecycle,NamespaceExists,LimitRanger,SecurityContextDeny,ResourceQuota/g" /etc/kubernetes/apiserver

按照顺序启动服务

systemctl start etcd docker kube-apiserver kube-controller-manager kube-scheduler kubelet kube-proxy

获取pause容器

先从docker hub上获取一个谷歌pause容器，然后修改镜像名并上传到私有仓库上
docker pull kubernetes/pause
docker tag docker.io/kubernetes/pause 172.16.1.150:5000/google_containers/pause-amd64.3.0
docker push 172.16.1.150:5000/google_containers/pause-amd64.3.0

确保安装了kubernetes的物理机可以从私有仓库上下载镜像

docker pull 172.16.1.150:5000/google_containers/pause-amd64.3.0

修改kubernetes配置文件

sed -i 's/KUBELET_ARGS=""/KUBELET_ARGS="--pod_infra_container_image=172.16.1.150:5000\/google_containers\/pause-amd64.3.0"/' /etc/kubernetes/kubelet

重启相关服务

systemctl restart etcd docker kube-apiserver kube-controller-manager kube-scheduler kubelet kube-proxy

第二章 实例练习

mysql

为mysql服务创建一个RC文件
mkdir -p /root/yaml/mysql
touch /root/yaml/mysql/mysql-rc.yaml
cat << EOF > /root/yaml/mysql/mysql-rc.yaml
apiVersion: v1
kind: ReplicationController
metadata:
  name: mysql
spec:
  replicas: 1
  selector:
    app: mysql
  template:
    metadata:
      labels:
        app: mysql
    spec:
      containers:
      - name: mysql
        image: mysql
        ports:
        - containerPort: 3306
        env:
        - name: MYSQL_ROOT_PASSWORD
          value: "123456"
EOF

master节点执行：
kubectl create -f /root/yaml/mysql/mysql-rc.yaml
将mysql-rc发布到kubernetes中；
查看创建的RC
kubectl get rc

第三章 基本概念

kubernetes: 是基于容器技术的分布式架构方案。
service: 服务。（拥有唯一的名字；拥有虚拟IP和端口号；可提供远程服务；被映射到提供这种服务能力的一组容器应用上。）
Pod: (内部运行Pause容器和业务容器，业务容器共享Pause容器的网络栈和数据卷)

Master: 运行了kube-apiserver;kube-controller-manager;kube-scheduler
Node:    运行了kubelet;kube-proxy

RC(replication controller): (目标Pod的定义；目标Pod的运行副本数；要监控的Pod的标签)
