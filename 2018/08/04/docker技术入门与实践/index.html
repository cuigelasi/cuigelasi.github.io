docker技术入门与实践

发表于 2018-08-04

第一章 初识docker

Docker引擎的基础是Linux容器技术。
LXC（linux containers）

第二章 docker的核心概念和安装

核心概念

docker镜像：类似虚拟机的镜像，是docker引擎的只读模板，包含了文件系统。
docker容器：类似轻量级的沙箱，docker用容器运行和隔离应用。
docker仓库：类似代码仓库，docker集中存放镜像文件场所。

第三章 镜像

搜寻镜像

docker search <image_name>

下载镜像

docker pull

查看镜像

docker images

查看镜像详细信息

docker inspect <image_id/image_name>

查看镜像指定内容

docker inspect -f {{".Architecture"}} <image_id/image_name>

删除镜像

删除镜像标签

docker rmi <image_name>

删除镜像id

docker rmi <image_id>

强制删除镜像

docker rmi <image_name/image_id>

创建镜像

容器保存为镜像

docker commit <container_name> <image_name>
-a 作者信息
-m 提交信息
-p 提交时暂停容器运行
docker commit -a "cgsl" -m "create image_name" <container_name> <image_name>

虚拟机直接保存为镜像

tar -cvpf /tmp/system.tar --directory=/ --exclude=proc --exclude=sys --exclude=dev --exclude=run --exclude=boot .

基于本地模板导入

cat <image_tar> | docker import - <image_name>

dockerfile制作镜像

docker build -t <image_name> .

保存镜像为tar包

docker save -o <image_name.tar> <image_name>

载入tar为镜像

docker load --input <image_name.tar>    
docker load < <image_name.tar>

修改镜像的标签

docker tag <image_tag_old> <image_tag_new>

上传镜像

dockers push

第四章 容器

创建容器

新建容器

docker create -it <image_name>

新建并启动容器

docker run -itd <image_name>
-i 让容器的标准输出持续打开
-t 让docker分配一个伪终端，并绑定到容器的标准输出上
-d 让容器在后台以守护态形式运行
--name 设置容器名
-h 设置容器内的主机名
--privileged 打开路由等系统内部文件的修改权限
--device 指定连接系统设备
/usr/sbin/init centos7容器需要此参数管理systemctl
/bin/bash centos6容器需要此参数保持容器持久化开启
--ip 指定ip
--network 指定docker网络

容器启停

启动容器

docker start <container_id>

停止容器

docker stop <container_id>

重启容器

docker restart <container_id>

查看容器

查看详细信息
docker ps -a
查看容器id
docker ps -aq

删除容器

-f 强制终止并删除运行中的容器
-l 删除容器连接，但保留容器
-v 删除容器挂载的数据卷
删除指定容器
docker rm -f -v <container_name/container_id>
删除所有容器        
docker rm -f -v $(docker ps -aq)

进入容器

attach

docker attach <container_name/container_id>

exec

docker exec -it <container_name/container_id> /bin/bash

nsenter

获取pid
docker inspect -f {{.State.Pid}} <container_name/container_id>
通过nsenter进入容器
nsenter --target <Pid> --mount --uts --ipc --net --pid

导出容器

docker export <container_name/container_id> ><container_name.tar>

导入容器

将tar文件导入成镜像
cat <container_name.tar> | docker import - <image_name>

第五章 仓库

公有仓库

docker hub 公有仓库
注册docker hub；
登陆docker hub；
使用docker hub。

私有仓库

搭建仓库

解决docker push http 问题

（Docker从1.3.X之后默认docker registry使用的是https，所以当用docker pull命令下载远程镜像时，如果远程docker registry是非https的时候就会报错误。）
vi /etc/sysconfig/docker
OPTIONS='--selinux-enabled --insecure-registry <eth0_ip>:5000' 
其中--insecure-registry <eth0_ip>:5000是自己添加的。

重启docker服务

systemctl restart docker

下载registry，并启动，通过-v参数将镜像文件放在指定的本机路径下

sudo docker run -d -p 5000:5000 -v /opt/data/registry:/tmp/registry registry --restart=on-failure:5
本机路径是/opt/data/registry

上传镜像

查看本地有哪些镜像

curl -X GET http://<eth0_ip>:5000/v2/_catalog

修改镜像tap

docker tag <image_tag> <eth0_ip>:5000/<image_tag>

上传镜像到本地仓库

docker push <eth0_ip>:5000/<image_tag>

继续查看本地有哪些镜像

curl -X GET http://<eth0_ip>:5000/v2/_catalog

删除镜像

【删除远程仓库容器】
私有仓库的镜像无法正常删除
首先要进入到registry容器中
docker ps -a
docker exec -it <registry容器id> sh
修改config.yml，使得允许删除
vi /etc/docker/registry/config.yml


直接删除需要删除的本地仓库镜像 （删除的是元数据
rm -rf /var/lib/registry/docker/registry/v2/repositories/<镜像名>
查看此时本地仓库的大小
du -sch /var/lib/registry/
进行垃圾回收（会删除层数据，不建议删除）
registry garbage-collect /etc/docker/registry/config.yml

配置连接镜像仓库

sed -i "/{}/d" /etc/docker/daemon.json
echo {"insecure-registries":["<eth0_ip>:5000"]} >> /etc/docker/daemon.json    
systemctl restart docker

第六章 数据管理

数据卷

数据卷是一个可供容器使用的特殊目录，数据卷可以在容器间共享和重用，对数据卷的修改会立马生效，对数据卷的更新不会影响镜像，卷会一直存在直到没容器使用。
-v 可以在容器内创建一个数据卷
docker run -itd -v /<catalogue>:/<catalogue> <image_name> /bin/bash
docker run -itd -v /<catalogue> /bin/bash

挂载一个主机目录作为数据卷

docker run -itd -v /<catalogue>:/<catalogue> <image_name> /bin/bash

挂载一个本地主机文件作为数据卷

docker run -itd -v /<filename>:/<filename> <image_name> /bin/bash

挂载数据卷容器

docker run -itd -v /<catalogue> --name <containner_data_name> <image_name> /bin/bash
docker run -itd --volumes-from <container_data_name> --name <container_name> <image_name> /bin/bash

利用数据卷容器迁移数据

备份

docker run -itd -v /<catalogue1> --name <containner_data_name1> <image_name> /bin/bash
docker run --volumes-from <container_data_name1> -v $(pwd):/backup --name <container_name1> <image_name> tar cvf /backup/backup.tar /dbdata

恢复

docker run -itd -v /<catalogue2> --name <containner_data_name2> <image_name> /bin/bash
docker run -itd --volumes-from <containner_data_name2> -v $(pwd):/backup --name <container_name2> <image_name> tar xvf /backup/backup.tar

第七章 网络基础配置

端口映射

-p 指定端口
-P 随机端口 (hostport:containerport)
docker run -itd -P <image_name>
docker run -itd -p 8080:80 <image_name>
## 容器互联
docker run -itd --name <container_name1> --privileged <image_name> /usr/sbin/init
docker run -itd --name <container_name2> --privileged --link <container_name1>:<container_name1> <image_name> /usr/sbin/init

第八章 Dockerfile创建镜像

Dockerfile基本结构

四大部分：基础镜像信息、维护者信息、镜像操作指令、容器启动时执行的执行指令。
vim common/Dockerfile

vim common/start.sh

vim common/common.sh

指令

FROM 指定是基于什么基础镜像创建
MAINTAINER 指定维护者信息
RUN 镜像操作指令
CMD 指定启动容器时执行的命令
EXPOSE 指定容器暴露的端口
ENV 指定环境变量
ADD 指定dockerfile内目录复制
COPY 指定本机目录复制
ENTRYPOINT 指定容器启动之后执行的命令
VOLUME 指定创建数据卷进行挂载
USER 指定运行时的用户名
WORKDIR 指定工作的目录
ONBUILD 指定以此次生成的镜像为基础镜像，再创建镜像时执行的命令
