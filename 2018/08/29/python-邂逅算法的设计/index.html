python 邂逅算法的设计

发表于 2018-08-29 | 分类于 算法

邂逅是多么浪漫的事情，但是在现实生活中邂逅总是那么遥不可及，有人说邂逅不可求但是可以科学的设计。

准备

设计邂逅算法，我们首先要准备好僚机。假设你有三个室友，让他们作为你的僚机部署在女神必经的三条路上，当他们看到你的女神就立马微信联系你。这里我们梳理一下，你的三个僚机室友所处的位置为三个点，所以点位数是3。当僚机室友遇见了你的女神，他才是完成了使命的僚机，我们命名为有效点位。话说如果室友见到一个人就给你发一次消息，我们则定义你收到所有消息的次数是总次数，那其中含有女神的消息为有效消息。

整理一下，我们现在有四个参数：总点位数，总次数，有效点位数，有效次数。

设计

当一切准备妥当之后，我们现在就缺算法设计了，设计好算法就可以实现邂逅啦。

算法设计中权重必不可少，我们现有的四个参数其中总点位数和总次数是相同组数据，有效点位数和有效次数是相同组数据。相同组数据需要设置权重，于是我们设计点位权重是0.6，次数权重是0.4。还没结束哦，一般来说算法要符合实际，没有百分百的可能性，于是我们添加一个衰减因子（0.9），让我们的算法无论怎么算都不会超过百分百。

算法的设计思想其实很简单：在有限区域内，女神出行必然选择区域内的某条行走路线，通过算法推断女神与你邂逅的概率。一般来说，我们的总点位要足够的多，总次数要足够多，所以你需要准备很多的僚机，让他们多多汇报情况。如果你只能出动三位僚机，那么算法的误差必然存在，需要运气加持。

现在我列出我的邂逅算法吧，
$$
（点位权重有效点位数/总点位数 + 次数权重有效次数/总次数）*衰减因子
$$
算法已经设计好啦，现在要用实践来证明了！！！
