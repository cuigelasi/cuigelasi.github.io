每日解析（11.7 每日一题解析）

昨天的教程中留下了一个问题：如何在django资源创建后立马启动django工程

解析：

首先是分析问题，上篇教程中我们是创建了mysql和django的基础镜像，并且创建了其deployment和service资源。在现有的资源中，我们第一个缺少的是django project。所以第一个要解决的就是获取django project！其次django需要连接数据库使用，好在昨天我们已经确定了service的cluster ip，当我们获取到django project后修改setting文件之后，我们就可以设置启动django工程啦。

不用担心，接下来我们就按照解析的思路去实现这个功能~

Django + Git 镜像

既然我们想要获取django project，我立马就想到用git来实现，git 下载最新的django project。其实大家也可以通过搭建http服务器，wget下载django project，如果要做持续集成，建议大家使用git方式，因为我们可以通过gitlab+jenkins快速实现动态发布。

制作新的镜像时，我们可以在已经制作好的django镜像上添加git功能，形成新的镜像。同时为了考虑到django project的自动运行web服务，我们需要一个开机脚本，在镜像中设置开机初始化脚本~~
