Docker Skill 101 - Skill 006 - 类宿主机容器

环境背景：

Docker 中有一个争议的地方就是该不该将Docker容器当宿主机使用，也就是将Docker 容器当类宿主机容器使用。大部分人觉得是Docker技术的倒退，因为Docker设计之初就是Docker面向容器，并不运行多组进程。但是也有部分有志之士认为，一个新技术的推广需要一个最包容的切入点，让更多的人使用新技术，了解新技术。如果把Docker容器当作类宿主机使用，可以很快的将现在运行的服务和运用迁移到类宿主机容器中。

我个人觉得，好酒也怕巷子深，你总把一个很好的技术束之高阁，用自己的思想来定义技术的用法，本身就是错误的！黑猫白猫，抓到老鼠就是好猫！只要技术能够快速投入生产，带来便利，为啥不先尝试使用，在使用的过程中，更深入的了解。最后的最后，我相信大家在使用Docker类主机之后，会慢慢的切换思想，改造自己的容器。

提出问题：

制作一个类似正常宿主机的镜像，可以运行多个进程和服务

解决方案：

  • 模拟一台宿主机的镜像
  • 安装部署所需要的应用程序

实际操作：

第一步：

创建一个类宿主机容器，我选用最基础的centos镜像

第二步：

在宿主机容器上安装ssh服务，httpd服务，nfs服务

第三步：

启动容器，检查各个服务状态

总结

我们公司在使用Docker技术的时候，也是从类宿主机容器出发，一步步让研发、测试、运维、实施了解并使用Docker技术，成功在公司内部推广了Docker技术，并且随着时间的推移，大家频繁使用中，一步步优化容器结构，重新设计。所以新技术的推广，一个切入点是非常重要的。
