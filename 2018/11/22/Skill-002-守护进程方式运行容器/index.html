Docker Skill 101 - Skill 002 - 守护进程方式运行容器

环境背景：

守护进程，说的直白点就是说，在后台运行的特殊进程 。如果不使用守护进程方式运行容器的话，那么伪终端结束的时候容器也将退出之时。在实际生产环境下，我们有很多的容器，需要稳定的运行，即使伪终端结束也要保证容器能够正常运行，只要守护进程存在那么容器将一直存在。

提出问题：

以守护进程方式在后台运行一个Docker容器

解决方案：

启动容器的时候通过-d的参数，让Docker容器在后台以守护进行形式运行

实际操作：

docker容器和普通的进程本质上是一致的，默认都是在前台运行。进程在后台运行我们可以用nohub &来操作，但是很明显这也不是很方便的，每启动一个容器被迫使用nohub &。最烦人的是会出现一个不停增长的日志，你必须要管理。显而易见，nohub & 不是我们理想的方式，所以我们还是使用守护进程方式来运行容器吧。

参数解析

-i 让容器的标准输入保持打开

-t 开启伪终端并绑定到标准输出上

-d Docker 容器在后台以守护态形式运行

-rm 进程结束之后便删除容器

–restart=no 重启策略：容器退出时不重启

–restart=always 重启策略：容器退出时总是自动重启

–restart=on-failure:3 重启策略：只在失败时候重启，重启三次
