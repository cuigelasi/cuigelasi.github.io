kubernetes 从入门到弃坑-3

发表于 2018-09-30 | 分类于 K8S

前言

安装kubernetes 时大家都会遇到一个问题，被墙了！由于谷歌的某些网址无法访问，而K8S又是基于这些网址下载一些资源，所以明明按照官方教程去安装，但是就是无法成功。有些人会说，直接翻墙出去啊，殊不知我们要是可以翻墙还会遇到这些问题啊。好了，今天我教大家一种不需要翻墙安装K8S的方法。

实验环境

centos7 系统

前方有坑

SElinux

我们为了容器能够与本机文件系统交互 需要关闭 SELinux。

桥接网络支持

centos7系统中我们需要打开桥接网络支持，不然部署之后你的K8S桥接网络是无法连通的。

SWAP

kubernetes要求必须关闭swap，所以我们还是默默关闭swap把，注意如果你是在生产环境中，装系统的时候就不要弄swap！！！

google源

我们安装kubeadm的时候，官方推荐的源是google的源，我们实际部署的时候千万记得修改源，不然一天都装不上的。

k8s 基础镜像

和google源是一样的，我们无法获取谷歌源，需要提前获取。

防火墙

K8S需要打开防火墙，内部用的是桥接网络通过iptable管理，不开防火墙通信也是不通的。

主节点

协议 方向 端口范围 目的 使用
TCP 入站 6443 Kubernetes API服务器 所有
TCP 入站 2379-2380 etcd服务器客户端API kube-apiserver，etcd
TCP 入站 10250 Kubelet API 自己，控制平面
TCP 入站 10251 KUBE-调度 自己
TCP 入站 10252 KUBE-控制器经理 自己

工做节点

协议 方向 港口范围 目的 使用
TCP 入站 10250 Kubelet API 自己，控制平面
TCP 入站 30000-32767 NodePort服务 所有

docker mirror

创建/etc/docker/daemon.json。如果你不修改下载镜像会很慢的。
