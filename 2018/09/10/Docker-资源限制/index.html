Docker 资源限制

发表于 2018-09-10 | 分类于 Base

限定CPU、内存、IO资源是因为防止某个容器占用太多资源而影响其他容器的性能，接下来我们从这三个方面讲解一些docker中限制那些事。

内存限制

容器可使用的内存分为物理内存和swap。

-m / –memory 设置内存

–memory-swap 设置内存加swap一起使用限制

设置内存200M，swap内存100M

对容器进行压力测试

–vm 1 启动1个内存工作线程

–vm-bytes 280M 每个线程分配280M

【注】

如果只使用-m / –memory，那么会默认分配和内存等量的swap

CPU限制

默认情况下，容器可以使用宿主机的CPU资源，我们可以使用命令设置CPU的使用限制

-c / –cpu-shares

这里的设置限制不是限制的具体数量而只是一个相对的权重

通过可以看到container_1使用cpu是container_2的两倍

IO带宽限制

限定IO磁盘读写，可以确保磁盘读写能够得到限制不会因为一个容器非正常读写导致其他容器无法进行读写操作

–blkio-weight 也是一个相对的权重

bps （byte per second）是每秒读写数据量

iops（io per second）每秒 IO 的次数

–device-read-bps 限制读某个设备的bps

–device-write-bps 限制写某个设备的bps

–device-read-iops 限制读某个设备的iops

–device-write-iops 限制写某个设备的iops

常见问题：

问：处理器权重怎么理解？是容器間的比重吗。那单个容器如何限制？比如我让docker上所有容器占用一半的cpu资源，可以实现么

答：可以的，需要先收集每个CPU的编号，然后容器启动时指定其只使用某个CPU或几个CPU

问：但这样实现不符合docker随处运行的理念，而且公司的服务器还好，如果是云服务器不可能收集到编号

答：设置 cpu-period 为 100ms，cpu-quota 为 200ms，表示最多可以使用 2 个 cpu。如果你只有四个CPU这样就可以限定只用一半了。
