OpenStack 核心组件深度学习 第一册 导读

发表于 2018-10-30
收费课程，请添加博主微信：cui15629135537

自我介绍

本人网名崔格拉斯，在云计算，大数据，安全行业摸爬滚打多年。 在14年自媒体大火的时候我带着一群乳臭未干的大学生跟随行业大佬做着初创公司，梦想着公司上市，升职加薪，走向人生巅峰。理想很丰满现实很骨感，新媒体运营和白领零食微店等项目并没有带来足够的收益，所以程序员出身的我们开始转型。OA系统，在线诊疗，警务大数据…在做这些项目的时候，遇到的痛点问题不仅仅是客户需求的多变，还有小型创业公司底层资源的匮乏。16年我毅然投入到了警务安全行业，专注于做云计算的底层云平台技术。从VMware vSphere 到OpenStack 私有云 公有云，再到现在的K8S 容器云 。深刻的感觉到云计算行业的学习曲线是非常陡峭的，难点不是部署一套成熟的私有云或者运用公有云，而是对私有云进行高可用架构设计和服务上公有云之后错误排查，这个时候你会发现自己没有透彻了解OpenStack技术。实践出真知，实际部署之后各种生产问题的浮现，会更让自己觉得OpenStack 的不简单，还需要不停的学习。

很多人遇到难啃的问题，总会放弃，但是总有一小部分人能够知难而上，而成功往往也属于这一小部分人。Openstack学习必然是一个艰难的事情，但是真正理解了各个OpenStack组件的功能以及组件间的相互协作流程，那么在日常工作中将会是一笔无形的财富。未来遇见了再多bug时，也不会慌，因为你知道每个组件负责什么工作，也知道每个组件是怎样去工作，排查思路了然于心。

我曾遇到过一种人，他能熟练的搭建云平台，能够天花乱坠的告诉我现有的架构是多么的好，遇到生产bug时却只能百度谷歌。我曾问他，你的Neutron Plugin为何只用OVS？他会义正言辞的告诉你，Open vSwitch 比Linux Bridge 稳定，性能好更加有优势 。但是你真的测过这两种Plugin的稳定性和性能吗？OpenStack Rocky稳定版都已经出来了，官方一直在推荐使用Linux Bridge！Open vSwitch与Linux bridge都有各自的适合的场景，对于用户来说这是两种优秀的网络解决方案，Open vSwitch 适合于SDN对集中管控的需求和更新更多的网络特性，但是在稳定性，大规模网络部署等场景中Linux bridge 更加适合。思维定势是可怕的，当学习了某个技术之后，一直认为自己掌握的技术是最先进最好的时候，就是被时代淘汰的时候。我希望每一个云计算从业人员或者爱好者能够花些时间去了解每个组件的运行原理，了解相关的技术知识，去实践部署去验证性能。

说到底，OpenStack也是一门开源技术，它时刻的更新着自身的源码和架构，那我们从事这个行业，时时刻刻都与这些新技术打交道，所以我们自身也要经常更新自己的知识。Keystone 大家都知道他是身份认证服务的组件，身份验证的流程，也都大概知晓。但是其实Keyston的验证方式早就已经变更过，现在使用Fernet Token验证，而你的认知是否还停留在Token验证？Fernet Token 和 之前 Token 相较而言优势在哪？为何OpenStack 认证服务全面使用 Fernet Token 而放弃 Token 。我希望大家带着这些疑问，在接下来的教程中获得一个可以让自己信服的答案。

目标读者

这套课程目标读者分为两类：

  1. OpenStack 初学者

    正是因为之前自己学习OpenStack的时候经历重重困难，所以更能了解一个深入浅出的剖析OpenStack教程，能够帮助初学者快速了解OpenStack 的组件知识，掌握组件核心工作模式，从而更深刻的认知OpenStack。赠人玫瑰手有余香，希望这个教程帮助初学者少走弯路，正式入门。

  2. OpenStack 运维开发工程师

    我在现在公司的职位是DevOps，所以编写这个教程的初衷也是有将自身DevOps的工作经验与大家分享。技术更新迭代，我们也要不停的回顾旧知识学习新知识，学习这个教程实则就是知识充电。期望在之后的工作中分析相关bug时，可以为大家启迪一些思路。

如何学习OpenStack

如何学习和了解OpenStack，那么其实就是了解一个开源软件。开源软件的学习一定不要因为一小段时间未理解其内部原理而萌发退意。 我们都知道 一万小时法则 ：在任何领域取得成功的关键跟天分无关，只是练习的问题，需要练习1万个小时。3年内，每天练习9小时，必然可以成功。道理都知道，能否坚持就要看个人了，俗话说师傅领进门修行靠个人。

OpenStack也不例外，所谓的OpenStack大牛，正是在你看不到的地方努力学习了一万小时才成为了大牛。如果你想节约点时间，你需要的是更勤奋的练习和前辈的经验。例如我的这么课程，虽然不敢说是什么OpenStack大牛，但是也是一名身经百战的一线DevOps，我的经验对于你来说很有借鉴的价值。

除此之外，我们还需要去开源软件的官方网站去寻找最新的资料。你如果可以熟练在OpenStack官方网站找到你需要的东西，那么就说明你对OpenStack已经比较熟悉。花费大量的时间阅读Openstack官网的资料，是一件很枯燥的事情，但好在我已经阅读了而且将读完的感受都记录在教程中，从我的教程中学习肯定比官网有趣味的多。

最后学习OpenStack需要一个氛围，你可以加入我的课程讨论群，与全国各地OpenStack从业人员一起探讨学习。当然如果你有时间和很好的英语能力也是可以通过国外论坛混国外的OpenStack圈子。
