kuberenetes 非敏感信息处理方案 configmap

与secret类似，我们如果想对非敏感数据进行特殊处理，可以使用configmap，这样我们就可以明文使用配置文件

1. configmap创建方式

总共有四种，其中前三种了解即可，重点是最后一种通过yaml文件创建。

方式一：–from-literal

kubectl create configmap myconfigmap –from-literal=config1=test1 –from-literal=config2=test2

一个–from-literal 参数对应一个键值对

方式二：–from-file

一个文件对应一个键值对

方式三：–from-env-file

env.txt 中的每一行对应一个键值对

方式四：yaml

myconfigmap.yml

kubectl apply -f myconfigmap.yml

2. 查看configmap

可以查看已存在的configmap

kubectl get configmap

DATA 代表有几个键值对

查看value

kubectl edit configmap myconfigmap

3. 在pod中使用configmap

方式一：volume foo

将需要的configmap键值对挂载到容器的/etc/foo目录下，便可直接使用，并且动态修改configmap的值，容器中也是能够动态更新的

myconfigmap_volume.yml

方式二： 环境变量

环境变量可以不用每次从文件中读数据，但是缺点是无法动态更新configmap的值

myconfigmap_env.yml

env 设定环境变量

name 环境变量名，注意要大写

valueFrom 键值对的value来源

configMapKeyRef 定义从configMap中获取value

name 选择的secret

key 键值对的key

4. yaml写配置文件

log_configmap.yml

data 定义配置文件

logging.conf 是配置文件名

| 用来分隔

以下四行是配置文件的内容

class: logging.handlers.RotatingFileHandler

formatter: precise

level: INFO

filename: %hostname-%timestamp.log

kubectl apply -f log_configmap.yml

kubectl get configmap

kubectl describe configmap myconfigmap

在pod中使用log的配置文件

mypod_log.yml
