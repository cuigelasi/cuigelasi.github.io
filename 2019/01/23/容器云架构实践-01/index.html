容器云架构实践-01

发表于 2019-01-23 | 分类于 容器云

前言

2018年是我在云计算领域深耕的第二年，成功的将docker技术在企业业务中落地实践，虽然这个过程中遇到了很多的问题，但是都一一披荆斩棘完美解决，所以18年是收获满满的一年。

新技术的应用，会引发更深层次的思考，深入了解Docker之后，你会自然而然的想去了解Docker的编排技术Kubernetes，当我深入了解K8S后，我的内心告诉我，“基于kubernetes做一个容器云吧！”。17年我深耕的是OpenStack云平台技术，18年深耕的是Docker技术，在19年用Kubernetes技术将所学的技术全部整合起来，做一个云原生的容器云将是一个很棒的idea。

说做就做！接下来我将记录我研究容器云的技术笔记，与大家一起分享从零开始制作一套云原生的容器云。

做什么?

做一件事情前，我们首先要了解我们做的事情是什么，并且能够把这件事情给一个新人直白的描述清楚。既然我们要做一套云原生的容器云，那么我们来一一阐述下把。

云原生？

Cloud Native 翻译成中文是云原生。这个词的含义很多很复杂。如果你想了解更多关于云原生的知识，需要自己去探寻。这里我只简单说一下我个人的理解。

了解云原生这个词，是因为学习Kubernetes。Kubernetes现在是CNCF (云原生应用基金会)主推的容器编排技术 解决方案，CNCF之所以这么大力推K8S，一方面是K8S技术的优益性，另一方面是K8S是CNCF自己管理的技术项目。这里不得不佩服谷歌，K8S前身是谷歌的Borg 容器编排管理平台，14年谷歌将Borg开源更名为Kubernetes，16年谷歌把Kubernetes项目控制权交给CNCF，18年CNCF宣布Kubernetes正式毕业。所以说谷歌是时刻布局，步步为营，比国内的宫斗剧都要精彩。

言归正传，云原生到底是什么呢？首先云原生是一种新的概念，它是一个集合包含了多种技术和管理方案。对于云计算来说，云原生是一系列的云计算、企业管理方法的集合。说点直白易懂的，云原生是DevOps、持续交付、微服务、敏捷开发、业务重组……。如果你还是没明白，那就记住，云原生就剩云计算的技术加上企业云时代管理方案的集合。

容器云？

容器云字面意思就是容器的云，实际也是如此它指的是以容器为单位，封装环境、提供构建、发布、运行分布式应用平台。也就是说，容器云是管理容器资源的云平台，在这个云平台上可以对容器资源进行调度管理。当我们理解了容器云是什么，我们也就知道我们需要对容器进行编排调度的工具来制作容器云。因此我们要使用现在行业内最权威的编排调度工具-Kubernetes来进行容器云开发。

Kubernetes?

kubernetes我们常说的K8S，Kubernetes 生于谷歌公司，是谷歌10多年容器编排调度经验的智慧结晶。前面我们也提到K8S的一些发展历程，2018不仅仅是K8S从CNCF毕业，同时也是K8S战胜了所有的编排调度工具，成为业内编排标准的一年。Kubernetes到底是什么，怎么去理解K8S，我们将在接下来的章节中，一一与大家进行分享。
